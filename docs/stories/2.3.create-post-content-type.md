# Story 2.3: Create "Post" Content Type in Strapi

## Status

Draft

## Story

**As a** content administrator,
**I want** to have a "Post" content type in Strapi for blog functionality,
**so that** I can easily create and manage blog content without developer assistance.

## Acceptance Criteria

1. "Post" content type is created in Strapi with fields for title, content, author, and image
2. Content type supports rich text editing for blog posts
3. Fields are properly validated and documented
4. Content administrators can create, edit, and publish blog posts
5. API endpoints for retrieving blog posts are functional
6. Draft/publish workflow is implemented and functional
7. Content type follows best practices for headless CMS design

  - [ ] Configure formatting options
  - [ ] Test rich text functionality

## Dev Notes

### Relevant Source Tree Info
In Strapi, content types are typically defined in:
- `src/api/post/` - Main directory for the content type
- `src/api/post/content-types/post/` - Schema definition
- `src/api/post/controllers/` - API controllers
- `src/api/post/services/` - Business logic
- `src/api/post/routes/` - API routes

### Testing
- Test content type creation in Strapi admin panel
- Verify all fields behave as expected
- Test rich text editing functionality
- Validate field validation rules
- Test content creation, editing, and publishing workflows
- Verify API endpoints return data in expected format
- Test draft/publish workflow functionality
- Test various query parameters for API endpoints
